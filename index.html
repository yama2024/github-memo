GitHubとは？なぜ重要なのか

GitHubの正体を理解しよう

GitHub = Git + Hub

  • Git: ファイルの変更履歴を自動で記録する「タイムマシン」システム
  • Hub: みんなでコードを共有・協力するためのオンライン「基地」

非エンジニアにとっての価値

安全性: コードが消失する心配がない
成長: AIと一緒にプログラミングスキルを向上
協力: 他の人との共同作業が簡単
公開: 作品を世界に発信できる

GitHubを一言で表すと

「コード専用のタイムマシン機能付きクラウドストレージであり、みんなで協力するための秘密基地！」

GitHub Copilot：AIペアプログラミングの革命

GitHub Copilotとは？

GitHub Copilotは、AIがあなたの"プログラミング相棒"となってくれるツールです。

  • リアルタイムコード提案: 書きたいコードを予測して提案
  • 自然言語での指示: 日本語でやりたいことを説明するだけ
  • エラー解決支援: バグの修正方法も教えてくれる
  • 学習効率向上: プログラミング学習が40%速くなる

学生なら無料で利用可能！

GitHub Educationに登録すれば無料！

GitHub Desktop：初心者に優しいGUI操作

GitHub Desktopをインストール

  1. 1
    GitHub Desktop公式サイトからダウンロード
  2. 2
    インストーラーを実行
  3. 3
    GitHubアカウントでログイン

GitHub Desktopのメリット

  • 視覚的操作: マウスクリックでGit操作が可能
  • 変更の可視化: ファイルの変更点が一目でわかる
  • 簡単なブランチ管理: ブランチの作成・切り替えが簡単
  • コンフリクト解決: 問題が起きても視覚的に解決

基本インターフェースの説明

  • 1. リポジトリリスト: 管理しているプロジェクト一覧
  • 2. ブランチ選択: 現在作業中のブランチ
  • 3. 変更ファイル一覧: 編集したファイルの表示
  • 4. コミット履歴: これまでの変更記録
  • 5. コミットボタン: 変更を記録するボタン

基本ワークフロー：Clone → Edit → Commit → Push

ステップ1：リポジトリを作成

  1. 1. GitHub.comで「New repository」をクリック
  2. 2. Repository name: プロジェクト名を入力（例：my-first-project）
  3. 3. Description: プロジェクトの説明
  4. 4. Public/Private: まずはPrivateを選択
  5. 5. Add README: 必ずチェックを入れる！
  6. 6. 「Create repository」をクリック

ステップ2：ローカルにクローン

GitHub Desktopを使用する場合：

  1. 1. リポジトリページで「Code」→「Open with GitHub Desktop」
  2. 2. 保存場所を選択
  3. 3. 「Clone」をクリック

手動でクローンする場合：

  1. 1. リポジトリのURLをコピー
  2. 2. GitHub Desktopで「Clone a repository」
  3. 3. URLを貼り付けて「Clone」

ステップ3：ファイルを編集

お気に入りのエディタ（VS Code、Cursorなど）でファイルを開き、自由に編集しましょう。

初心者におすすめの最初の編集：
# My First GitHub Project

こんにちは！これは私の最初のGitHubプロジェクトです。

## やりたいこと
- [ ] HTMLページを作成
- [ ] CSSでスタイリング
- [ ] JavaScriptで動きを追加

## 学習記録
- 2025/06/18: GitHubアカウント作成
- 2025/06/18: 最初のリポジトリ作成

ステップ4：変更をコミット

GitHub Desktopでのコミット手順：

  1. 1. 左側の「Changes」タブで変更されたファイルを確認
  2. 2. 各ファイルの隣にある✓マークをクリック（ステージング）
  3. 3. 下部の「Summary」に変更内容を記入
  4. 4. 「Commit to main」をクリック
良いコミットメッセージの例：

✅ 良い例：

  • • ホームページのレイアウトを修正
  • • ログイン機能を追加
  • • バグ修正：ボタンが押せない問題を解決

❌ 悪い例：

  • • 更新（何を更新したか不明）
  • • あああ（意味がない）
  • • test（テスト内容が不明）

ステップ5：GitHubにプッシュ

  1. 1. GitHub Desktopの上部にある「Push origin」をクリック
  2. 2. 初回は「Publish branch」と表示される場合があります
  3. 3. プッシュが完了すると、GitHub.comで変更が確認できます

この基本サイクルを覚えよう：

編集 → ステージング → コミット → プッシュ

GitHub Pages：無料でWebサイト公開

GitHub Pagesとは？

GitHubが提供する無料の静的サイトホスティングサービスです。HTMLやCSSで作ったサイトを世界に公開できます。

公開されるURL：

https://ユーザー名.github.io/リポジトリ名/

設定方法

  1. 1
    リポジトリの「Settings」をクリック
  2. 2
    左メニューの「Pages」を選択
  3. 3
    「Source」で「Deploy from a branch」を選択
  4. 4
    「Branch」で「main」を選択
  5. 5
    「Save」をクリック

独自ドメインの設定

  1. 1. ドメインを取得（お名前.comなど）
  2. 2. GitHubのPages設定で「Custom domain」に入力
  3. 3. DNSの設定でCNAMEレコードを追加

よくある問題と解決方法

エラー1：「Permission denied (publickey)」

原因: SSH設定が正しくない

解決方法:

  • HTTPS URLを使用する
  • またはSSH鍵を正しく設定する

エラー2：「merge conflict」

原因: 同じファイルの同じ行を複数人が編集

解決方法:

  1. GitHub Desktopの「Resolve conflicts」をクリック
  2. 手動でコンフリクトを解決
  3. 解決後、コミット

エラー3：「failed to push」

原因: リモートリポジトリが更新されている

解決方法:

  1. まずgit pullで最新状態を取得
  2. コンフリクトがあれば解決
  3. 再度git push

初心者向けトラブル回避策

  • 定期的にプル: 他の人の変更を定期的に取り込む
  • 小さなコミット: 大きな変更は分割してコミット
  • 意味のあるメッセージ: コミットメッセージは具体的に
  • バックアップブランチ: 重要な作業前にブランチを作成

次のステップ：さらなる活用方法

1. GitHub Actions：自動化の世界

CI/CDパイプラインを構築して、コードのテストやデプロイを自動化：

name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Setup Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '16'
    - run: npm ci
    - run: npm run build
    - name: Deploy
      uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./dist

2. GitHub Codespaces：クラウド開発環境

  • ブラウザ上で完全な開発環境を利用
  • セットアップ不要で即座に開発開始
  • 月60時間無料で利用可能

3. オープンソースプロジェクトへの貢献

  1. 1. 興味のあるプロジェクトをStar
  2. 2. Forkして自分のアカウントにコピー
  3. 3. 改善案を実装
